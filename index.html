Troubleshoot Offline Gateway

// when the gateway is not connected to remote.it
⚠️
If the gateway is not connected to Remote.it, SSH access is impossible until connectivity is restored.
Possible Causes
#CauseAction
1Gateway is powered downCheck physical power / PDU / breaker
2Network cable / WiFi disconnectedInspect Ethernet connection on the gateway
3Firewall blocking outbound trafficAsk IT to whitelist outbound TCP for Remote.it
4Hardware damage to gateway unitMay need a replacement unit shipped
Diagnostic Checklist
  • Confirm gateway shows OFFLINE in Remote.it — not just slow to respond
  • Ask on-site contact to check power LEDs on the gateway unit
  • Ask on-site contact to re-seat or replace the Ethernet cable
  • Check whether the customer recently changed firewall or router rules
  • If all checks pass — escalate for possible hardware replacement
