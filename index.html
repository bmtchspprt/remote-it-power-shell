Step 1 — Find Your Gateway

// remote.it device lookup
🔗
Open Remote.it Device Portal app.remote.it/#/devices → Search for your gateway
INSTRUCTIONS
  • Log in to app.remote.it/#/devices
  • In the search bar, type the gateway name (e.g. CompanyTX-00)
  • Locate the device in the list and click it to open its services
  • Find the SSH service — note the Public Endpoint (host:port)
ℹ️
The Online Status in Remote.it reflects a background ping. If it shows online, the gateway has responded to a ping and is reachable.

Step 2 — Build SSH Command

// paste the Remote.it public endpoint → get your PowerShell command
ENTER PUBLIC ENDPOINT

Copy the public endpoint from Remote.it and paste it below. Example: proxy16.rt3.io:37081

GENERATED POWERSHELL COMMAND
PS>ssh -p <port> binmaster@<host>
# Endpoint → Command mapping:
proxy16.rt3.io:37081  →  ssh -p 37081 binmaster@proxy16.rt3.io
⚠️
Remote.it endpoints are time-limited. If the connection drops, return to Remote.it and click the service again to get a fresh endpoint.

Step 3 — Connect & Verify

// open powershell and establish the ssh session
YOUR SSH COMMAND
PS>— enter endpoint in Step 2 —
CONNECTION STEPS
  • Open Windows PowerShell or Windows Terminal
  • Paste the SSH command above and press Enter
  • If prompted about host authenticity, type yes and press Enter
  • Enter the password when prompted
  • Confirm you see the gateway's shell prompt
PS C:\Users\You> ssh -p 37081 binmaster@proxy16.rt3.io

Step 4 — Run Gateway Commands

// once connected via ssh, use these commands
CommandPurposeNotes
bcsensors View sensor readings Also sends manual readings to BinCloud
sudo reboot Reboot the gateway Only via SSH — no other method
uptime Check how long gateway has been online Good first diagnostic
ping -c 4 8.8.8.8 Test internet connectivity Rules out network issues
df -h Check disk space Ensure storage isn't full
systemctl status remoteit Remote.it agent status Verify agent is running
QUICK COMMAND LAUNCHER
ℹ️
Gateway readings go directly to BinCloud. BinCloud pulls online status from Remote.it separately. The gateway never connects directly to BinCloud.

Step 5 — Troubleshoot Offline Gateway

// when the gateway is not connected to remote.it
⚠️
If the gateway is not connected to Remote.it, SSH access is impossible until connectivity is restored.
POSSIBLE CAUSES
#CauseIndicator / Action
1Gateway is powered downCheck physical power / PDU / breaker
2Network cable / WiFi disconnectedInspect Ethernet connection on gateway
3Customer firewall blocking outbound trafficRemote.it requires outbound TCP — ask IT to whitelist
4Hardware damage to gateway unitMay need replacement unit shipped
DIAGNOSTIC CHECKLIST
  • Confirm gateway shows OFFLINE in Remote.it (not just slow)
  • Ask on-site contact to check physical power LEDs on gateway
  • Ask on-site contact to re-seat the Ethernet cable
  • Check if customer recently updated firewall rules
  • If all above OK — escalate for possible hardware replacement

Step 6 — About Remote.it

// how the secure relay architecture works
HOW IT WORKS
LayerComponentRole
1Remote.it AgentInstalled on gateway; establishes secure outbound connection to relay servers
2Cloud Relay ServersIntermediary; routes traffic between you and the gateway — no inbound ports needed
3AuthenticationYou log in via web/app; Remote.it provides a unique, time-limited endpoint
4ConnectionYour SSH client → Remote.it relay → gateway agent — no public IP required

✅
No open inbound ports, no static IP, no VPN — Remote.it uses only outbound connections from the gateway, making it firewall-friendly and secure.
